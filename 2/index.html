Game Development

Unity 3D

During the 2nd term of the first year we were introduced to Unity 3D engine. This is the main development kit I use I have a project which is at alpha stage and I am currently working on several other projects on my own and in groups.


Microsoft XNA

In our main programming module in the first year we were using C# with the XNA library attached to create a simple game. The game was a hide and seek style game which implemented searching, seeking and random moving AI mechanics, along with a mechanics for hiding, shooting and moving using the player input.


Stencyl

I used stencyl works for the first term of my first year before I was competent enough in C# to start scripting in Unity We were made to use stencyl in order to teach us the development process instead of actually developing games. This proved useful when I came to make games on Unity. I made an action platformer that was a 2d mix between Super Mario and Luigi's Mansion for my stencyl game.

Software and System Development

Invoice Management System

As part of BC Studio I contributed to the development of an Invoice Management System for a Property Management firm in Leicester. The system deals with quotes and invoices for jobs that are created in the system. Each job has a number of tasks which are also programmed in. The previous system they used was developed in Microsoft Access and our client requested our system to be developed in Access as well. We used mostly just Access with queries and forms, however for some features I had to use my good old Visual Basic knowledge from my A level course. This was mixed in with some SQL as well and after a bit of a struggle merging VB, SQL and Access into one we finally got it working!


Fast Food Till System

For my A2 Computing Project I had to find a client and build them a system, since the till at my parents shop isn't computerized I thought it would be a good way to go. In Visual Basic I programmed a till system that dynammically creates the till setting reading too and from text files. Along side this we had to do a full documentation of the system and all of it's specs and design processess.

Version Control

Dropbox and Git

Last year I was mainly keeping version control by making copies of my work whenever I made changes, these would all be stored on dropbox so I could access them anywhere. This system became very inconvenient and spatially inefficient and so I was introduced by a friend to Git. I started off on the command line however I'm more recently using the GitHub gui for git, as GitHub gives a good student account.

Project Management

Postit Notes

Where Elliot is living in the room next door to mine any work we do as a team we have a postit note wall outside our room, we take the project and split it into tasks and place them in either the "Ry", "Elliot" or "Both" column depending on who's required to work on the task. I also have a wall in my room that I manage my life on using postit notes and find them incredibly useful to work with!

Pivotal Tracker

When it comes to doing work that postit notes can't manage I use pivotal tracker. Whether it's for my own personal projects or in a group I find it an incredibly useful way to keep track of projects and keep track of who's doing what. I find pivotal tracker great due to how easy it is to split the problems up into task and give each smaller task a timeframe instead of for the overall problem.


Wunderlist

When the two other options above aren't available, e.g. when I'm not sat at my computer desk. I have an app called Wunderlist on my iPad which is a "to-do list" app. I find this useful for when I'm out and about and if I think of something on a project whilst I'm out I can note it down on there and come back to it on the webapp on my pc to then transfer to either postit notes or pivotal!
